
# Инструкция по развёртыванию проекта StudPortrait

## Системные требования
- Windows / macOS / Linux с установленным Docker Desktop
- Docker Desktop должен быть ЗАПУЩЕН перед началом развёртывания

## Шаг 1. Получение проекта
Откройте терминал и перейдите в папку с проектом:
cd D:\Programms\codes\vuz_practice\stud-portrait

## Шаг 2. Запуск всех сервисов
Выполните команду в корне проекта:
docker-compose up -d --build

Что произойдёт после выполнения:
- Docker соберёт образы для всех трёх сервисов
- Запустятся контейнеры: база данных, бэкенд, фронтенд
- База данных инициализируется из файла database/init.sql

## Шаг 3. Проверка работоспособности
Откройте браузер и перейдите по адресам:
Фронтенд (React): http://localhost:3000
Бэкенд API (Django): http://localhost:8000
Админка Django: http://localhost:8000/admin

Проверка через командную строку:
docker ps                    # показать запущенные контейнеры
docker-compose logs          # показать логи всех сервисов
docker logs studportrait_backend   # логи только бэкенда

## Шаг 4. Остановка сервисов
Обычная остановка:
docker-compose down

Полная остановка с удалением данных базы данных:
docker-compose down -v

## Возможные проблемы и решения
1. Ошибка "порт уже занят" - закройте программы, использующие порты 3000, 5432, 8000
2. Бэкенд не видит базу данных - проверьте файл backend/src/studportrait/settings.py: в секции DATABASES параметр HOST должен быть 'database'
3. Docker не запускается на Windows - запустите Docker Desktop от имени администратора
4. Пустая страница фронтенда - нажмите F12 в браузере → вкладка Console → посмотрите ошибки

## Примечания
- Пароль для базы данных в режиме разработки: postgres
- При изменении кода требуется пересборка: docker-compose up -d --build
- Все сервисы работают в изолированной сети Docker

## Критерии выполнения
1. Все сервисы запускаются одной командой: docker-compose up -d --build
2. Учтены аспекты безопасности: база данных не публикует порт наружу, используется internal network
3. Инструкция позволяет развернуть проект на чистом устройстве без дополнительных действий
